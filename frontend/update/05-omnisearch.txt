Changing `QuoteRequestItem` descriptions, morphing the `specifications` from strings into arrays, and renaming core fields to `wholesalePrice` and `stockLevel` across the board just to make the backend stop crashing the frontend. It definitely looks like a "make it work now, make it elegant later" situation.

The beauty of moving to an **Omni-Search Catalog** is that it actually *decouples* your UI from that messy backend taxonomy. Instead of building a fragile UI sidebar that relies on deeply nested database categories, you just point a single, powerful search bar at the `/api/parts` endpoint. When you eventually revert or refactor the database schema, this UI won't break because it's just passing a text string to the backend!

Here is the complete, ultra-clean code for your new **Omni-Search Command Center**. You can use this to completely overwrite your `frontend/app/catalog/page.tsx`.

### The New `frontend/app/catalog/page.tsx`

```tsx
'use client';

import { useState, useEffect, useMemo } from 'react';
import { Search, ShoppingCart, CheckCircle2, AlertCircle } from 'lucide-react';

// Using the exact contract established in BACKENDREPORT Entry 3
interface InventoryItem {
  skuId: string;
  partName: string;
  category: string;
  specifications: { label: string; value: string }[];
  wholesalePrice: number; // in cents
  stockLevel: number;
  qualityGrade: string;
}

export default function OmniCatalogPage() {
  const [activeBrand, setActiveBrand] = useState<'Apple' | 'Samsung'>('Apple');
  const [searchQuery, setSearchQuery] = useState('');
  const [parts, setParts] = useState<InventoryItem[]>([]);
  const [isLoading, setIsLoading] = useState(true);

  // In a real app, this would hit your /api/parts?device= endpoint
  // Fetching all parts for the active brand to allow instant client-side filtering
  useEffect(() => {
    async function loadCatalog() {
      setIsLoading(true);
      try {
        // Replace with your actual apiFetch wrapper if needed
        const res = await fetch(`${process.env.NEXT_PUBLIC_API_URL || 'http://localhost:3001'}/api/parts`);
        const json = await res.json();
        if (json.success) {
          setParts(json.parts);
        }
      } catch (error) {
        console.error("Failed to fetch catalog:", error);
      } finally {
        setIsLoading(false);
      }
    }
    loadCatalog();
  }, []);

  // The Omni-Search "Magic" Filter
  const filteredParts = useMemo(() => {
    return parts.filter((part) => {
      // 1. Filter by top-level brand toggle (assuming skuId or partName contains the brand info)
      // In a real scenario, your API might return a 'brand' field, but we'll use partName here
      if (!part.partName.toLowerCase().includes(activeBrand.toLowerCase())) return false;

      // 2. If no search query, show all for that brand
      if (!searchQuery.trim()) return true;

      // 3. AND-logic omni-search: user types "15 pro battery" -> must match all words
      const terms = searchQuery.toLowerCase().split(' ').filter(Boolean);
      const searchIndex = `${part.skuId} ${part.partName} ${part.category} ${part.qualityGrade}`.toLowerCase();
      
      return terms.every(term => searchIndex.includes(term));
    });
  }, [parts, activeBrand, searchQuery]);

  return (
    <div className="min-h-screen pt-24 pb-16 px-4 sm:px-6 lg:px-8 max-w-7xl mx-auto animate-in fade-in duration-500">
      
      {/* 1. The Brand Toggle (Primary Universe Filter) */}
      <div className="flex justify-center mb-12">
        <div className="inline-flex p-1 bg-card border border-border rounded-sm shadow-sm">
          {['Apple', 'Samsung'].map((brand) => (
            <button
              key={brand}
              onClick={() => setActiveBrand(brand as 'Apple' | 'Samsung')}
              className={`px-8 py-2.5 text-sm uppercase tracking-widest font-bold transition-all rounded-sm ${
                activeBrand === brand 
                  ? 'bg-primary text-primary-foreground shadow-sm' 
                  : 'text-muted-foreground hover:text-foreground hover:bg-secondary/50'
              }`}
            >
              {brand}
            </button>
          ))}
        </div>
      </div>

      {/* 2. The Command Search Bar */}
      <div className="max-w-3xl mx-auto mb-16 relative group">
        <div className="absolute inset-y-0 left-0 pl-6 flex items-center pointer-events-none">
          <Search className="h-6 w-6 text-muted-foreground group-focus-within:text-primary transition-colors" />
        </div>
        <input
          type="text"
          autoFocus
          placeholder="Search generation, part, or SKU (e.g., '15 Pro Battery')..."
          value={searchQuery}
          onChange={(e) => setSearchQuery(e.target.value)}
          className="w-full bg-card/80 backdrop-blur-sm border-2 border-border text-foreground px-16 py-6 text-xl focus:outline-none focus:border-primary focus:ring-1 focus:ring-primary rounded-sm transition-all placeholder:text-muted-foreground shadow-sm"
        />
        <div className="absolute -bottom-6 right-0 text-[10px] text-muted-foreground font-mono uppercase tracking-widest">
          {isLoading ? 'Syncing...' : `${filteredParts.length} Results Found`}
        </div>
      </div>

      {/* 3. The Live Data Grid */}
      {isLoading ? (
        <div className="flex justify-center py-24">
          <div className="w-8 h-8 border-2 border-primary border-t-transparent rounded-full animate-spin"></div>
        </div>
      ) : (
        <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-6">
          {filteredParts.map((part) => {
            const isContactForPrice = part.wholesalePrice === 0;
            
            return (
              <div 
                key={part.skuId} 
                className="bg-card/90 backdrop-blur-sm border border-border p-5 flex flex-col justify-between rounded-sm hover:border-primary/50 hover:shadow-md transition-all group"
              >
                <div className="mb-6">
                  <div className="flex justify-between items-start mb-4">
                    {/* Golden SKU */}
                    <span className="font-mono text-[10px] text-primary bg-primary/10 border border-primary/20 px-2 py-1 uppercase tracking-widest rounded-sm">
                      {part.skuId}
                    </span>
                    {/* Stock Status */}
                    <span className={`flex items-center gap-1 text-[9px] font-mono uppercase tracking-wider px-2 py-1 rounded-sm ${part.stockLevel > 0 ? 'text-success bg-success/10 border border-success/20' : 'text-destructive bg-destructive/10 border border-destructive/20'}`}>
                      {part.stockLevel > 0 ? <CheckCircle2 className="w-3 h-3" /> : <AlertCircle className="w-3 h-3" />}
                      {part.stockLevel > 0 ? `${part.stockLevel} IN STOCK` : 'OUT OF STOCK'}
                    </span>
                  </div>
                  
                  <h3 className="text-base font-bold text-foreground leading-tight mb-2 group-hover:text-primary transition-colors cursor-pointer">
                    {part.partName}
                  </h3>
                  
                  <p className="text-xs text-muted-foreground uppercase tracking-wider">
                    {part.qualityGrade} Grade • {part.category}
                  </p>
                </div>

                <div className="flex justify-between items-end pt-4 border-t border-border mt-auto">
                  <div>
                    <span className="block text-[9px] text-muted-foreground uppercase tracking-widest mb-1">Wholesale</span>
                    <span className={`font-mono font-bold ${isContactForPrice ? 'text-sm text-warning' : 'text-lg text-foreground'}`}>
                      {isContactForPrice ? 'CONTACT' : `$${(part.wholesalePrice / 100).toFixed(2)}`}
                    </span>
                  </div>
                  
                  <button 
                    disabled={part.stockLevel === 0}
                    className="bg-secondary text-secondary-foreground hover:bg-primary hover:text-primary-foreground border border-border hover:border-primary px-4 py-2 text-[10px] font-bold uppercase tracking-widest transition-all disabled:opacity-50 disabled:cursor-not-allowed rounded-sm flex items-center gap-2"
                  >
                    <ShoppingCart className="w-3 h-3" />
                    Add
                  </button>
                </div>
              </div>
            );
          })}
        </div>
      )}

      {/* Empty State */}
      {!isLoading && filteredParts.length === 0 && (
        <div className="text-center py-24 bg-card border border-border rounded-sm">
          <Search className="w-12 h-12 text-muted-foreground mx-auto mb-4 opacity-50" />
          <h3 className="text-lg font-bold text-foreground mb-2">No components found</h3>
          <p className="text-sm text-muted-foreground">Try adjusting your search terms or clearing the brand filter.</p>
        </div>
      )}
    </div>
  );
}
```

### Why this is a massive UX Upgrade:
1. **The "Engineering Grid" Shines:** Because the main search bar and the product cards use `bg-card/80 backdrop-blur-sm`, they are slightly translucent. This means the subtle blueprint grid background we built earlier will physically show *through* the cards, creating an incredible glass-like depth.
2. **Frictionless Navigation:** Notice there is no "Search Button." The `filteredParts` `useMemo` hook calculates the AND-logic search in real-time as they type. It feels as fast as native desktop software.
3. **Data Integrity Built-In:** It perfectly handles the `wholesalePrice === 0` ("Contact for Price") rule established in `BACKENDREPORT` Entry 1, and expects prices to arrive in cents.
